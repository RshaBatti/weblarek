Стек: HTML, SCSS, TS, Vite

Структура проекта:
- src/ — исходные файлы проекта
- src/components/ — папка с JS компонентами

Важные файлы:

# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент за отображение, которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.

Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.

## Данные

### Интерфейс товара (IProduct)
Описывает структуру данных для товара в каталоге магазина. Используется для отображения товаров, работы с корзиной и передачи данных в API.
interface IProduct {
  id: string;
  description: string;
  image: string;
  title: string;
  category: string;
  price: number | null;
}

### Интерфейс покупателя (UserProfile)
Описывает данные покупателя, необходимые для оформления заказа. Используется при валидации данных, передаче информации в API и отображении в интерфейсе.
interface UserProfile {
  payment: TPayment;
  email: string;
  phone: string;
  address: string;
}

## Модели данных

#### Класс TotalItems
Класс относится к слою Model (данные) и предназначен для управления каталогом товаров, доступных в магазине, а также для отслеживания товара, выбранного пользователем для детального просмотра.

Конструктор класса не принимает параметров.

Поля класса:  
`private items: IProduct[]` - массив объектов типа IProduct. Содержит полный список товаров, загруженных с сервера.
`private preview: IProduct | null` - хранит объект товара, который в данный момент открыт в модальном окне. Если окно закрыто, значение — null.

Методы класса:  
`setItems(items: IProduct[]): void` - записывает полученный массив товаров в поле items. Используется при первичной загрузке данных из API.
`getItems(): IProduct[] `- возвращает актуальный массив всех товаров, хранящихся в модели.
`getProductById(id: string): IProduct | null` — ищет товар в массиве items по указанному id. Возвращает объект товара типа IProduct, если найден, или null, если товар не существует.
`setPreview(item: IProduct): void` — устанавливает объект товара в поле preview. Это инициирует событие открытия модального окна в интерфейсе.
`getPreview(): IProduct | null` — возвращает текущий объект товара из поля preview. Если превью не задано, возвращает null.
#### Класс BasketItems
Класс отвечает за хранение выбранных пользователем товаров и управление логикой их покупки. Позволяет динамически изменять состав корзины и вычислять финансовые показатели заказа.  

Конструктор класса не принимает параметров.

Поля класса
`private items: IProduct[]` — массив объектов типа IProduct. Хранит перечень всех товаров, которые пользователь добавил в корзину.  

Методы класса
`add(product: IProduct): void` - принимает объект товара и добавляет его в конец массива items. После добавления инициирует событие обновления состояния корзины.
`remove(id: string): void ` - ищет товар в массиве по его идентификатору (id) и удаляет его. Если в корзине несколько одинаковых товаров, удаляется только один экземпляр.
`getCount(): number ` - возвращает текущую длину массива items (общее количество единиц товара в корзине).
`clear(): void` — очищает массив items, удаляя все товары из корзины. Используется после успешного оформления заказа или по запросу пользователя.
`getItems(): IProduct[]` - возвращает актуальный массив всех товаров, находящихся в корзине, для их последующего рендеринга в интерфейсе.
`getTotalPrice(): number` - рассчитывает суммарную стоимость всех товаров в массиве items. Если цена товара не указана (null), она считается равной 0.
`isInBasket(id: string): boolean` - проверяет, содержится ли в массиве items товар с указанным id. Возвращает true, если товар найден, и false в противном случае (используется для изменения состояния кнопок в каталоге).  

#### Класс IBuyer
Класс относится к слою Model. Он отвечает за хранение контактных данных покупателя, адреса доставки и выбранного способа оплаты, а также за проверку корректности этих данных перед отправкой заказа.  

Конструктор класса не принимает параметров.

Поля класса
`private payment: TPayment`— выбранный способ оплаты.
`private email: string` — контактный e-mail пользователя.
`private phone: string` — контактный номер телефона.
`private address: string `— адрес доставки товара.

Методы класса
`setData(data: Partial<UserProfile>): void` — записывает значения в соответствующие поля класса (например, при вводе данных в форму). Принимает объект с одним или несколькими полями из интерфейса UserProfile.
`getData(): UserProfile` — возвращает объект, содержащий все текущие данные покупателя для последующей передачи в API при оформлении заказа. Возвращаемый объект соответствует интерфейсу UserProfile.
`validate(): UserProfileValidationErrors`— выполняет валидацию всех полей модели по следующим правилам:
- payment не должен быть null;
- email не должен быть пустой строкой;
- phone не должен быть пустой строкой;
- address не должен быть пустой строкой.
Возвращает объект с ошибками валидации типа UserProfileValidationErrors. Если все поля валидны, возвращается пустой объект {}.
`clearData(): void` — сбрасывает все поля в исходное состояние (null или пустая строка) после успешного завершения заказа.

### Слой коммуникации

#### Класс CommunicationApi
Отвечает за взаимодействие с сервером «вебларёк»: получение данных (каталог товаров) и отправку данных (информация о заказе).
Конструктор:
`constructor(api: IApi)` - конструктор инициализирует экземпляр CommunicationApi, сохраняя переданный объект api в приватном свойстве. Это позволяет классу выполнять HTTPзапросы через указанный APIклиент.

Поля класса:  
`private api: IApi` — экземпляр класса Api, предоставляющий методы get и post

Методы класса
`fetchProducts(): Promise<FetchData>` -  выполняет GET‑запрос на эндпоинт /product/ для получения каталога товаров с сервера.
Возвращаемое значение: Promise<FetchData> — промис, который при успешном выполнении разрешается объектом типа FetchData.
Логика работы:
вызывает метод api.get('/product/');
преобразует ответ сервера в объект типа FetchData;
при ошибке: логирует сообщение в консоль через console.error();
пробрасывает ошибку дальше с помощью throw.
Исключения: выбрасывает ошибку при сетевом сбое или некорректном ответе сервера.
`sendOrder(orderData: IOrder): Promise<OrderResponse>` - выполняет POST‑запрос на эндпоинт /order/ для отправки данных о заказе на сервер.
Параметры:
`orderData (IOrder)` — объект с данными заказа, содержащий:
payment, email, phone, address — данные покупателя (наследуются от UserProfile);
total — общая стоимость заказа;
items — массив идентификаторов товаров (string[]).
Возвращаемое значение: Promise<OrderResponse> — промис, который при успешном выполнении разрешается объектом типа OrderResponse.
Логика работы:
вызывает метод api.post('/order/', orderData);
преобразует поле data из ответа сервера в объект типа OrderResponse;
при ошибке: логирует сообщение в консоль через console.error();
пробрасывает ошибку дальше с помощью throw.
Исключения: выбрасывает ошибку при сетевом сбое, некорректных входных данных или ошибке на стороне сервера.
